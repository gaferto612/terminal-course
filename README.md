## 📖 About this course

This is a free, open-source course designed for **complete beginners** who have never touched a terminal — and want to become confident command-line users across every major shell.

You'll learn:

- 🐧 **Bash** — the universal Unix shell (Linux, macOS, WSL)
- 🪟 **PowerShell** — Microsoft's object-oriented shell
- 📟 **CMD** — the classic Windows command prompt
- 🐟 **Fish** — the user-friendly modern shell
- ⚡ **Zsh** — the macOS default and customization king
- 🐧 **WSL** — Linux inside Windows
- 🔧 **Plus**: Git, SSH, package managers, Python CLI, and more

## 🎯 Who this is for

- Complete beginners who feel intimidated by the "black box with text"
- Self-taught developers filling gaps in fundamentals
- Students preparing for IT, DevOps, or development careers
- Anyone who's tired of clicking through menus when one line would do

**No prerequisites.** If you can open an app on your computer, you can do this course.

## 🗺️ Course structure

The course is built in **7 parts** with **32 modules**. Each module includes:

- 📝 A written lesson (Markdown)
- 🎨 Diagrams and infographics (SVG)
- 🧪 Hands-on exercises
- ✅ Solutions to check your work
- 🃏 A quick-reference cheatsheet

| Part | Topic | Modules |
|------|-------|---------|
| **0** | [Getting Started](modules/00-getting-started/) | Setup & installation |
| **1** | [Foundations](modules/01-foundations/) | What is a shell? Mental models |
| **2** | [Bash Core](modules/02-bash-core/) | Navigation, files, pipes, processes |
| **3** | [PowerShell](modules/03-powershell/) | Objects, cmdlets, the Windows way |
| **4** | [Other Shells](modules/04-other-shells/) | Zsh, Fish, WSL, CMD |
| **5** | [Scripting](modules/05-scripting/) | Writing your own programs |
| **6** | [The Toolbox](modules/06-toolbox/) | Git, SSH, package managers, Python |
| **7** | [Productivity](modules/07-productivity/) | Customization, dotfiles, tmux |

## 🚀 Getting started

1. **Read the [Getting Started](modules/00-getting-started/README.md) module** to set up your terminal
2. **Follow the modules in order** — they build on each other
3. **Do the exercises** — reading isn't enough; you must type the commands
4. **Use the [cheatsheets](cheatsheets/)** as quick references
