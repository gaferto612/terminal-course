# Contributing to the Terminal Course

Thank you for considering a contribution! This course thrives on community input.

## Ways to contribute

- **Fix typos or unclear explanations** — these matter more than you think
- **Add diagrams** — visual learners benefit massively
- **Translate** — open an issue if you'd like to start a translation
- **Add exercises** — more practice problems are always welcome
- **Report broken commands** — terminal commands change with OS versions

## How to contribute

1. Fork this repository
2. Create a branch: `git checkout -b improve-bash-module`
3. Make your changes
4. Test any commands you add on the target OS
5. Submit a pull request with a clear description

## Style guide

- Write for **complete beginners**. If you use a term, define it first.
- Show **commands in code blocks** with the shell specified:
  ```bash
  ls -la
  ```
- Always show **expected output** so learners know what success looks like
- Prefer **diagrams** over walls of text for any structural concept
- Use **second person** ("you'll see...") and a warm tone

## Diagram standards

- SVG only (no PNG/JPG for diagrams)
- Use the color palette defined in `assets/css/style.css`
- Keep them readable at small sizes
- Excalidraw exports are welcome

## Code of conduct

Be kind. Beginners ask "basic" questions because they're learning. Patience is a feature.

## Questions?

Open an issue with the `question` label.
