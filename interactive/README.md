# 🧪 Try It — Interactive practice

You can practice without installing anything. These free, browser-based environments give you a real terminal in your browser.

## 🎯 Best for this course

### Killercoda (recommended)

Free interactive Linux terminals in your browser. No signup needed for many scenarios.

👉 [killercoda.com/playgrounds](https://killercoda.com/playgrounds)

Pick the **Ubuntu** playground and you're in a real Bash terminal in 5 seconds.

### Replit

Sign up free, then create a "Bash" repl. You get a persistent Linux environment that saves your work.

👉 [replit.com](https://replit.com)

### Play with Docker

Free 4-hour Linux sessions. Good for trying out commands without commitment.

👉 [labs.play-with-docker.com](https://labs.play-with-docker.com)

## 🪟 For PowerShell practice

### Try .NET PowerShell (browser-based)

👉 [tio.run/#powershell](https://tio.run/#powershell)

For a quick PowerShell one-liner test. Not a full shell session, but useful for syntax testing.

### Use GitHub Codespaces (with PowerShell installed)

If you have a GitHub account, every repo can spin up a free Codespace (a full VS Code environment in your browser). PowerShell Core works there.

## 🐟 For Fish, Zsh, and others

Most online terminals run Bash. To try Fish or Zsh in a sandbox:

- Use Killercoda's Ubuntu playground, then `sudo apt install fish` or `sudo apt install zsh`
- Or use [JSLinux](https://bellard.org/jslinux/) for a full Linux VM in your browser

## 🎮 Game-based learning

These turn shell commands into puzzles:

- **[OverTheWire Bandit](https://overthewire.org/wargames/bandit/)** — a series of SSH-based puzzles that teach Linux commands. Highly recommended for learning by doing.
- **[Vim Adventures](https://vim-adventures.com/)** — learn Vim through a 2D game (first few levels are free).
- **[CodeCombat](https://codecombat.com/)** — broader programming game that includes some shell concepts.

## ⏰ A suggested practice schedule

After each module:

1. Open Killercoda (or your local terminal)
2. Try every command from the module once
3. Do the exercises in the module's README
4. Pick **one** thing from that module to actually use this week in your real workflow

Repetition is the only path to fluency. **Type the commands.** Don't just read them.
